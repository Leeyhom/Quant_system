# 个人量化交易系统项目启动文档

## 1. 个人背景
- **编程能力**：有扎实的代码基础，熟悉 Python，有深度学习项目经验（如模型训练、数据处理、模型部署等）。
- **金融知识**：仅进行过简单的股票买卖操作，了解“买入”和“卖出”的基本概念。
- **量化知识**：几乎为零，不清楚量化交易的定义、常见策略、回测框架以及风控等概念。
- **学习目标**：希望在和 Claude Code 的交互过程中，边学边做，逐步构建一个可运行的量化交易程序。

## 2. 项目目标
- 搭建一个 **多市场量化交易程序**，初期以学习为核心，逐步实现在 A股、港股、美股 的自动化或半自动化交易，追求长期稳定收益。
- 程序需要具备完整的生命周期：**数据获取 → 策略开发 → 回测验证 → 模拟/实盘交易 → 监控与风控**。

## 3. 功能需求概览（渐进式）
### 第一阶段：学习与数据基础
- 学习量化交易基本概念（如 Alpha、Beta、夏普比率、最大回撤等）。
- 学习市场微观结构基础知识（A股T+1、涨跌停，港股与美股的交易规则等）。
- 搭建数据获取管道：至少覆盖 A股、港股、美股 的日线与分钟线数据（优先免费或低成本数据源）。

### 第二阶段：策略开发与回测
- 从经典策略学起：双均线、动量、均值回归、简单的机器学习预测模型等。
- 构建回测引擎，能评估策略的历史表现，输出绩效报告（收益率、回撤、夏普比率等）。
- 考虑真实交易约束（手续费、滑点、流动性、涨跌停限制等）。

### 第三阶段：执行与风控
- 对接券商 API（如盈透、富途、老虎等，或 A股券商如华泰、中信等）。
- 实盘前的模拟盘运行。
- 实现基本风控模块：仓位管理、最大回撤熔断、黑名单机制等。

## 4. 技术偏好
- 语言：Python（优先）
- 深度学习框架：PyTorch / TensorFlow（已在项目中积累经验，可用于特征提取或预测模型）
- 数据存储：本地 Parquet/CSV 或轻量数据库（SQLite）
- 交互方式：本工程通过 Claude Code 协助编码与学习，每个模块希望在理解原理后再动手实现，而非直接复制粘贴代码。

## 5. 交互要求
- **循序渐进**：请从我当前的知识水平出发，先解释必要概念，再引导编码。
- **解释驱动的编码**：每段代码请附带“为什么这么做”以及“在量化中的意义”。
- **阶段性总结**：每完成一个模块，请帮我梳理已掌握的知识点和后续学习路线。
- **代码风格**：追求可读性与模块化，便于后续扩展和维护。
