# 19 · 聚宽 A股 v6：从对齐版走向收益版

> 接 `18` 和 `jointquant/v4-ssot/v4_ssot_alpha_diagnosis.md`。v4-ssot 的价值在于三方口径对齐，
> 但它不是收益最大化版本。v6 的目标是提高 alpha 密度，同时保留可在聚宽实盘环境复现的简单逻辑。

## 一、v4-ssot 的核心问题

v4-ssot 聚宽导出显示：2019-2025 总收益 +123.18%，年化约 +12.6%，alpha 0.067，
最大回撤 27.09%。它有 alpha，但不够强，且 2021-2023 三年恢复能力差。

诊断后的优先级：

1. 行业映射没有覆盖扩展池：152 只里 38 只落入“其他”，污染行业中性化和行业上限。
2. 当前跑的是 `INCLUDE_HOLDER=False` 的 5 因子对齐版，不是收益版。
3. 价值因子重复投票：`1/PE、1/PCF、1/PS` 各占一份，组合仍像低估值轮动。
4. 没有趋势过滤，容易买入“便宜但仍在下跌”的价值陷阱。
5. 6 万本金、100 股整手约束下，现金拖累不可忽视。

## 二、v6 设计

v6 不做复杂机器学习，先修结构：

- `value_blend`：先把 `earnings_yield/cashflow_yield/sales_yield` 三个价值信号合成一票。
- `growth_peg`：继续使用 PEG 倒数，保留高覆盖成长信号。
- `amihud`：流动性冲击，越流动越好。
- `holder_concentration`：股东户数环比变化取负，作为筹码集中度信号；聚宽字段覆盖不足时自动跳过。
- `quality_roe`：半权重，避免单纯便宜股。
- `low_vol_60`：半权重，压波动和价值陷阱。
- `mom120 > -10%`：只做过滤，不做追涨。

组合参数：

```text
TOP_N = 10
INDUSTRY_CAP = 2
REBALANCE_DAYS = 60
MAX_EXPOSURE = 95%
MOMENTUM_120_MIN = -10%
```

重要 nuance：v4 旧版行业上限 2 失败，是因为它同时把持仓压到 6 只，集中度太高。
v6 保持 10 只，并补上趋势过滤和行业映射，行业上限 2 反而提高机会集，不再等价于 v4 的失败机制。

## 三、本地验证

脚本：

```bash
PYTHONPATH=. python scripts/joinquant_v6_validation.py
```

口径：

- 2019-01-01 ~ 2025-12-31
- 本地 cached `DEFAULT_POOL`，89 只有效票
- 6 万本金
- 100 股整数手
- 佣金最低 5 元、印花税、过户费、滑点

结果：

| 版本 | 参数 | 总收益 | 年化 | Sharpe | 最大回撤 |
|---|---|---:|---:|---:|---:|
| v4-ssot 5F baseline | top10 / 40d / cap1 | +285.34% | +22.15% | 1.04 | 19.55% |
| v5 baseline | quality_half / top10 / 60d / cap1 / mom>-10% | +314.08% | +23.46% | 1.17 | 14.19% |
| v6 default | value_blend / top10 / 60d / cap2 / mom>-10% | +338.22% | +24.50% | 1.29 | 15.23% |

v6 相对 v5：收益、年化、Sharpe 都提高，回撤略高约 1.3pct。考虑目标是“把收益做大、alpha 做高”，这个折中值得进入聚宽回测。

## 四、诚实记录

最初的直觉是“价值三因子冗余，应彻底降权并加入质量/低波/holder”。第一轮验证显示：
过度去价值化反而跑不赢 v5。A股这段样本里，价值暴露虽然冗余，但确实贡献收益。

最终 v6 采用折中：价值三因子先合成一票，避免 60% 价值权重；但行业上限从 1 放到 2，
让价值机会仍能被选中。收益来自“降低价值重复投票 + 放宽同行业机会 + 趋势过滤”三者共同作用。
