# 14 · 扩历史 + 正交因子 + 等权合成：项目第一个稳定的多因子超额

> 接 `13`。M13 用 ICIR×多切分把多因子合成的工程链路打通了，但在 **2 年(2024~2025)** 数据上任何合成都跑不赢单因子 `earnings_yield`，诊断结论写得很清楚：**瓶颈是信息量（历史太短 + 正交因子不足），不是加权方法**。M14 就按这个诊断对症下药，并得到了项目第一个**稳定跑赢的多因子组合**。

---

## 一、两个动作，对症 M13 的诊断

M13 §8 留下的两条改进路径，M14 一次做完：

**动作 1：扩历史到 2018**（`scripts/refetch_history.py` + `config.HISTORY_START`）
2 年只覆盖 1~2 个 regime，撞上 2025Q3 行业回调一次就被带偏，滚动只有 **4 个窗口**——统计上根本不可信。实测新浪行情、东财估值接口都能回溯到 **2018-01**。重拉后全池 1942 个交易日（≈7.8 年）、89 只，覆盖 2018 熊 / 2019~2021 结构牛 / 2022 熊 / 2023 震荡 / 2024~2025，滚动窗口从 4 增到 **23**。

> 缓存是「优先读本地」的，旧缓存只到 2024，所以必须用 `refetch_history.py` **强制覆盖重拉**，否则面板还是旧的 2 年。脚本顺带报告每票起始日、上市晚于 2019 的票（幸存/上市偏差提示）。

**动作 2：零成本补正交因子**（`factor/factors.py` 新增三个，无新数据源）
关键洞察是**现有估值面板里已经藏着「价值之外」的维度**，靠会计恒等式即可导出：

| 新因子 | 公式 | 经济含义 | 与价值正交吗 |
|------|------|---------|------------|
| `quality_roe` | **PB / PE** = ROE | 盈利能力（用净资产赚钱的效率） | 是（与 B/P 相关 −0.46，几乎反相位） |
| `growth_peg` | 1 / PEG | 相对盈利增速的便宜度 | 是（与价值相关 0.02） |
| `cashflow_yield` | 1 / PCF | 现金流相对市值（盈利的现金验证） | 部分（与价值相关 0.22） |

`PB/PE = ROE` 这一步值得停一下：PB=价格/每股净资产，PE=价格/每股收益，相除价格抵消，得每股收益/每股净资产 = **净资产收益率**。所以**质量因子根本不用新数据，已有的 PB、PE 一除就是**。实测全池 ROE 中位 ≈10%（p10 4%、p90 20%），数值合理。

## 二、长历史下，先看清每个因子的「regime 脸谱」

扩史后逐年 IC（双中性后）第一次让我们看清因子的 regime 依赖：

```
factor             2018   2019   2020   2021   2022   2023   2024   2025
earnings_yield   -0.007 -0.042 -0.059 -0.019 +0.019 +0.126 +0.033 +0.062
quality_roe      +0.020 +0.056 +0.077 -0.042 -0.047 -0.049 -0.030 +0.058
growth_peg       +0.038 -0.073 +0.040 +0.020 +0.047 +0.074 +0.016 +0.002
cashflow_yield   +0.017 -0.027 +0.030 -0.026 +0.001 +0.085 +0.030 -0.015
amihud           -0.009 +0.069 +0.084 +0.013 -0.025 -0.034 -0.029 +0.045
```

两个关键观察：
1. **价值（earnings_yield）2018~2021 是负的、2022~2025 才转正**——成长牛市里价值没用，价值回归后才发力。如果只有 2 年样本恰好骑在某一段，结论必然偏。
2. **quality_roe 与 value 几乎反相位**：value 失灵的 2018~2020，quality 正好有效；value 发力的 2022~2024，quality 转负。相关 −0.46 印证了这一点。

这种「你弱我强」的反相位，**理论上正是分散 regime 的理想搭档**——但下一节会看到，怎么合成它们是成败关键。

## 三、决定性结果：等权合成跑赢，IC 加权反而更差

滚动 walk-forward（train 480 / test 120 / step 60，**23 个跨 regime 窗口**，分层多头 L5 vs 等权全持有基准）：

| 策略 | 跑赢基准比例 | 中位超额夏普 | 全样本收益/夏普 |
|------|------|------|------|
| 单因子 earnings_yield（最优单因子） | 70% | +0.20 | +103% / 0.60 |
| 单因子 cashflow_yield | 61% | +0.16 | — |
| 单因子 growth_peg | 65% | +0.11 | — |
| **ICIR×多切分 IC 加权合成** | **39%** | **−0.09** | — |
| **等权合成（固定正向）** | **83%** | **+0.37** | **+251% / 0.90** |
| 等权全持有基准 | — | — | +148% / 0.69 |

**等权合成在跑赢比例（83% vs 70%）和超额夏普（+0.37 vs +0.20）上同时超过最优单因子，且回撤更低（26.3% vs 29.4%）。这是项目第一个稳定的多因子超额。**

而 IC 加权合成（M13 精心打磨的 ICIR×多切分）只有 39%，**比单因子还差**。为什么？

---

## 四、为什么等权 > IC 加权：「方向锁定陷阱」

这是 M14 最重要的方法论教训，和直觉相反，所以要讲透。

IC / ICIR 加权要用 **train 段的 IC** 来锁定每个因子的**方向**（IC<0 就反向）**和权重**。这在因子方向稳定时没问题，但 A股因子**方向会随 regime 翻号**：

- quality_roe 与 value 反相位。某个窗口的 train 段如果在「value 占优」期，算出 quality 的 IC 是负的 → 加权器把 quality **反向**；可到了 test 段 regime 切换成「quality 占优」，quality 实际是正贡献，但你押的是反向——**正好押反**。
- 两个反相位因子在静态加权下，权重方向都按 train 锁死，到 test 期一个押对一个押反，贡献**相互抵消**，分散不成反而内耗。

**等权 + 固定正向先验**绕开了这个陷阱：它不试图预测「这段该押哪个方向」，而是给每个**先验为正**的因子固定正权重。某个因子这段失灵（IC 暂时转负）顶多是贡献小，但不会因为「锁定了过期方向」而**反向押注**。换句话说：

> IC 加权是「赌我能预测哪个因子下一段有效」——在强 regime 切换下这个预测本身不可靠；等权是「我不预测，雨露均沾」——放弃了择时收益，换来了不被择错方向反噬的稳健。**当方向本身不可预测时，不预测就是最优策略。**

这再次印证项目第一性原理：**信息量优先于模型复杂度**。长历史（信息）+ 正交因子（信息）+ 最朴素的等权（最低复杂度），赢过了精巧的 IC 加权。

## 五、一个诚实的前提：等权只对「方向先验稳」的因子有效

等权不是万能药。它的前提是**分量的方向先验要稳（全样本 IC 净正向）**。demo 里 `select_stable_positive` 就在做这个筛选：

- `quality_roe` 虽然正交，但全样本 posRate 仅 50%、ICIR +0.04——**方向像抛硬币**（它在本段是反相位漂移的）。把它等权纳入只会稀释信号，故**剔除**。
- 最终纳入等权的是 earnings_yield / cashflow_yield / growth_peg / amihud 四个**净正向**因子。

这一步筛选基于**长历史 + 经济先验的「因子库选择」**（类似「决定研究哪些因子」），不是逐窗调参，门槛公开可复现，故不构成前视。

**另一个诚实的 nuance**：在 40 只小池上跑，筛选只留下 2 个因子、等权优势就不明显（61%），反倒 IC 加权偶然更高。原因是小池每层股票太少、分层噪声大。**等权分散的优势在全池（89 只、每层~18 只）才稳定显现**——这也说明池子规模本身是信息量的一部分。

## 七、一句话回顾

> M13 诊断「瓶颈是信息量不是加权方法」，M14 对症：**扩历史到 2018**（regime 覆盖：4 窗口→23）+ **零成本补正交因子**（ROE=PB/PE、1/PEG、1/PCF）。结果是项目第一个稳定多因子超额——**等权合成 83% 跑赢、超额夏普 +0.37**，且**等权 > IC 加权**：因为 A股因子方向随 regime 翻号，IC 加权锁定的旧方向会押反、反相位因子相互抵消，而等权+固定正向先验绕开了这个「方向锁定陷阱」。前提是只纳入方向先验稳的因子。下一步走向组合资金管理 / 模拟盘。

## 八、稳健性消融：83% 是结构性优势，不是单组超参的运气

83% 来自**一组**窗口设置（train480/test120/step60、再平衡20、5层），单组结果永远有「恰好挑到好参数」的嫌疑。`scripts/ablation_demo.py` 做**单轴消融**——固定基线，每次只改一个维度，看等权的优势对该维度是否敏感。

| 配置 | 等权 跑赢/超额夏普 | 最优单因子 | IC加权 | 等权占优 |
|------|------|------|------|------|
| 基线 (480/120/60, reb20, 5层) | **83% / +0.37** | 70% / +0.20 | 43% / −0.17 | ✓ |
| 短窗 (360/90/45) | 71% / +0.32 | 59% / +0.08 | 35% / −0.17 | ✓ |
| 长 train (600) | 81% / +0.38 | 76% / +0.20 | 48% / −0.16 | ✓ |
| **不重叠 step (90)** | **87% / +0.35** | 60% / +0.14 | 40% / −0.08 | ✓ |
| 再平衡 10（月内） | 74% / +0.31 | 61% / +0.05 | 35% / −0.21 | ✓ |
| 再平衡 30（季频） | 83% / +0.37 | 61% / +0.16 | 48% / −0.13 | ✓ |
| 分层 3（每层多） | 57% / +0.17 | 52% / +0.23 | 43% / −0.07 | ✓ |
| 分层 10（每层少） | 70% / +0.28 | 61% / +0.15 | 61% / +0.09 | ✓ |

**汇总：8 组设置 100% 跑赢基准过半、100% ≥ 最优单因子、100% ≥ IC加权、100% 超额夏普>0（中位 +0.33）。**

三个值得记的观察：
1. **最强的是「不重叠 step」(87%)**——这是最干净的检验（窗口互相独立、无重叠样本，重叠窗口会人为放大稳定性）。独立窗口下优势反而最大，说明它是真的。
2. **最弱的是「分层3」(57%)**——分层越粗（每层股票越多→越接近基准本身），选股 alpha 被稀释，等权与单因子差距收窄（57% vs 52%），但仍 ≥ 单因子。这是可解释的衰减，不是反例，印证了 §五「池子规模/每层股票数本身是信息量」。
3. **IC加权 8 组里 7 组超额夏普为负**——再次坐实「方向锁定陷阱」。

> 结论：M14 的等权优势在窗口长度、再平衡频率、分层粒度三个维度上都稳健，**是结构性的，不是超参运气**。地基确认后，下一步按计划接入港股/美股同步验证（最终目标：美股稳定收益），再做组合资金管理。
