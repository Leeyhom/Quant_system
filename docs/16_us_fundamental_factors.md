## 一、数据源：找到了比预期好得多的季报源

Stage 2a 当时以为美股基本面「只能从季报艰难拼点状面板，且 `stock_us_valuation_baidu`
已失效」。本次重新探查 akshare，发现一个**远比预期好的源**：

```python
ak.stock_financial_us_analysis_indicator_em(symbol="AAPL", indicator="单季报")
```

实测优点：
- **回溯到 2000 年**，单只拉取 ~0.3s；池中 ~80 只大多可用。
- **自带 `NOTICE_DATE`（公告日）**——这是季报防前视的命门。
- 字段丰富：ROE_AVG（质量）、毛利率/净利率、营收同比/净利同比（成长）、单季 EPS、负债率。

`stock_us_valuation_baidu` 仍然失效（确认）。所以**价值因子靠 EPS 自算**：
`earnings_yield = TTM_EPS / 价格`（= 1/PE）。

## 二、踩到一个关键数据质量陷阱（不修就会让回测严重劣化）

把季报拼成「日期×股票」面板时，第一反应是按 `NOTICE_DATE` 对齐——但实测发现东财的
公告日有个系统性问题（4 只票一致验证）：

```
REPORT_DATE   NOTICE_DATE   gap(天)
2024-12-28    2026-01-30    398   ← 老季度的公告日晚打戳约一年！
2025-03-29    2026-05-01    398
2025-06-28    2025-08-01     34   ← 最近 ~4 个季度才是合理的 ~34 天
2025-09-27    2025-10-30     33
```

**原因**：东财每次披露财报时，同时列出「本季 + 去年同期对比」两行，并把**去年同期那行**
也打上**本次**披露的 NOTICE_DATE。结果除最近 ~4 季外，老季度的公告日被晚标了约一年。

直接用它**不会造成未来函数**（偏晚=偏保守），但会让基本面信息**滞后近一年**，
严重劣化回测。修正（`us_fundamental_loader._NOTICE_CAP_DAYS=60`）：

```
effective_notice = min(原始 NOTICE_DATE, REPORT_DATE + 60 天)
```

- 最近季度 gap≈30 < 60 → 保留真实公告日；
- 老季度 gap≈398 > 60 → 回退到 report+60（美股 10-Q/10-K 法定披露窗口内，**绝不早于
  真实披露 ⇒ 无未来函数**），把人为的一年滞后压回到合理的两个月。

实测修正后 `effective_notice` **严格单调递增、无重复公告日**（顺带解决了重复戳问题）。

## 三、防前视面板：对齐键 = 公告日（不是报告期）

`build_us_fundamental_panels`（panel.py）仿 A股 `build_value_panels` 的 ffill 思路，
但**对齐键换成公告日**：把每条季报的指标值放在它的 `notice_date` 那天，reindex 到
交易日 index 后 **ffill**（公告后、下次公告前沿用最近一次已公告值），首个公告日前为 NaN。
TTM EPS 在 loader 内按 report_date 排序滚动求和（`rolling(4).sum()`），再绑定到该季公告日。

脚本里设了一道**防前视 gate**（首跑前人工核对），实测：

```
[gate] 抽查 AAPL @ 2021-12-29: eps_ttm(面板)=5.67
       面板值来自 report_date=2021-09-25 notice_date=2021-11-24 eps_ttm(源)=5.67
       公告日 ≤ 交易日? ✅ 无未来函数
```

## 四、美股专属费用模型（区别于 A股比例换手成本）

用户指出美股每笔交易有平台费。A股回测用 `cost_rate`（比例×换手金额），但美股券商
更常见**每股费 + 每笔最低费**（如 $0.005/股、最低 $1/笔）。两者在低价股/小额交易时
差异巨大——比例成本会严重**低估**真实费用。

`quant/backtest/us_cost.py`：`make_layered_cost_fn()` 产出一个 `cost_fn` 回调，把权重
变动换算成名义股数后套 `max(每股费×股数, 每笔最低费)`，再转成收益拖累。`layered_backtest`
新增 `cost_fn=` 注入点（**默认 None=保持 A股比例成本，向后兼容**）。

## 五、结果：项目首个在美股稳定跑赢的因子来自基本面

全样本 IC（horizon=20，全池 ~72 只有基本面数据）：

```
factor          meanIC   ICIR    t       posRate   解读
value_ey       +0.007   +0.05  +2.04    49%       价值（mega-cap 池里更强，见下）
quality_roe    +0.005   +0.04  +1.62    52%       质量
growth_rev     +0.020   +0.09  +4.20    52%       成长（营收同比）✅净正向
growth_profit  +0.010   +0.06  +2.52    57%       成长（净利同比）✅最稳
```

**与 Stage 2a 量价的鲜明对比**：量价因子里强信号全是负的（regime beta，|t|>7），
`select_stable_positive` 筛出 **0 个**；基本面因子**全部 meanIC 为正**，方向先验
与经济直觉一致，是真正可重复的 alpha 雏形。

滚动 walk-forward（train480/test120/step60，24 窗口，含美股费用）：

```
                  跑赢比例   中位超额夏普
growth_profit       71%       +0.22    ← 全池最优单因子
quality_roe         58%       +0.06
value_ey            50%       -0.10
（Stage 2a 量价最优 reversal 仅 29% / -0.14，对比鲜明）
```

全样本分层多头净值（含美股费用）：

```
            收益       夏普     回撤
等权合成    +487%     +1.00    29.18%   ← 收益更高、回撤显著更低
最优单因子  +554%     +1.04    38.15%   (growth_profit)
等权基准    +344%     +0.90    41.21%
```

等权合成跑赢基准 **+143pct 收益、回撤低 12pct**——首次在美股拿到稳定超额。

## 六、诚实的 nuance：方向先验随池子构成漂移

一个值得记录的观察：**小池（30 只 mega-cap）里价值因子极强**（value_ey t+5.99、
跑赢 67%），但**全池（~72 只）纳入更多股票后价值减弱、成长接力**。这导致
`select_stable_positive` 在全池只筛出 1 个净正向因子（growth_rev），等权合成因此
**未超过最优单因子**（等权 50% vs growth_profit 71%）。

这与 A股 M14 的教训呼应：等权合成的优势依赖「多个方向先验都稳的正交因子」。美股大盘池
里基本面因子虽然**全为正**（比量价好太多），但**单个因子的稳定性会随池子构成变化**——
小池里价值主导（低估值的成熟巨头跑赢），大池里成长主导。这不是方法问题，是**池子本身的
信息结构**。下一步方向：①扩池纳入中小盘（基本面分化更大，价值/成长信号更分明）；
②用上**做空**能力（多空对冲，把方向漂移的影响对冲掉）——这正是用户指出的美股相对 A股的
关键新能力，留作 Stage 2c。

> 一句话：**docs/15 的预言兑现了——稳定信号确实在基本面**。美股基本面因子全部方向正确、
> 单因子滚动跑赢 71%、等权多头收益+487%回撤仅29%，是项目首个在美股稳定跑赢的结果。
> 剩下的是把「方向漂移」用扩池或做空进一步驯服。
