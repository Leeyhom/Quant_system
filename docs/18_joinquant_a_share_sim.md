# 18 聚宽 A股模拟盘接入

本阶段目标不是继续优化样本内收益，而是把策略放进聚宽模拟盘的真实执行环境中，观察订单、成交、涨跌停、停牌、费用和现金拖累。

## 当前配置

- 本金：60000 元
- 持仓：6 只
- 总仓位：95%
- 调仓：每 60 个交易日
- 执行时间：14:30
- 未成交处理：留现金，不递补追买
- 佣金：万 2.5，最低 5 元
- 印花税：万 5，卖出单边
- 过户费：万 0.1，聚宽无独立过户费字段，策略中已并入买卖双边佣金
- 推送：飞书机器人

## 文件

聚宽策略文件：

```text
scripts/joinquant_cn_sim_strategy.py
```

把该文件完整复制到聚宽模拟交易策略编辑器中即可。该文件是自包含版本，不依赖本地 `quant/` 包。

## 数据口径

聚宽模拟盘运行在云端，使用聚宽自己的行情和基本面接口，不再使用本地 akshare。

本地项目继续承担：

- 研究和回测
- 第三方 RQAlpha 校验
- 参数扫描
- HTML 看板

聚宽承担：

- 云端定时运行
- 模拟盘订单撮合
- 持仓/现金/成交状态
- 每日飞书推送

## 第一版执行原则

第一版有意保持克制：

- 涨停不买
- 跌停不卖
- 停牌不交易
- ST、*ST、退市风险股过滤
- 买不到不递补下一名
- 未成交资金留现金

这样做的目标是减少执行逻辑噪音，先验证策略信号在真实模拟盘下是否还能保持可接受表现。
