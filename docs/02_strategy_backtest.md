# 02 · 第一个策略（双均线）与最小回测引擎

> 接 `01_concepts.md`。这一章我们用 M1 拉到的日线数据，写一个能跑的策略，并亲手搭一个回测引擎，算出收益率/最大回撤/夏普。

---

## 一、什么是「策略」？信号 vs 仓位

一个策略本质上回答一个问题：**「现在我该持有多少这只股票？」**

我们把它拆成两层，职责清晰：

1. **策略层**：只负责产出「**目标仓位信号**」。最简单的形式是 0/1：
   - `1` = 满仓持有
   - `0` = 空仓（不持有）
   - 它**不关心**钱、手续费、能不能成交——只表达「我想持有还是不想」。
2. **回测层**：拿着信号，结合真实约束（手续费、T+1、滑点）去模拟「真的这么做会怎样」，算出账户净值。

> 这样分层的好处：换策略只改策略层，回测引擎完全复用；测真实约束的影响只改回测层。

---

## 二、双均线策略（Dual Moving Average）

**直觉**：均线是「最近 N 天收盘价的平均」，代表一段时间的成本/趋势。
- **短期均线**（如 5 日，MA5）反应快，贴近近期价格。
- **长期均线**（如 20 日，MA20）反应慢，代表中期趋势。

**规则**：
- 当 **MA5 上穿 MA20**（短线抬头超过长线）→ 认为趋势向上 → **信号 = 1（买入/持有）**
- 当 **MA5 下穿 MA20** → 趋势走弱 → **信号 = 0（卖出/空仓）**

这就是经典的「金叉买、死叉卖」。它是一个**趋势跟随**策略：不预测顶底，只在趋势确立后跟上。

⚠️ **一个致命的新手错误：未来函数（look-ahead bias）**
今天的均线要等今天收盘才算得出来。如果你用「今天的信号」去「今天收盘前」交易，等于偷看了未来。
正确做法：**信号要往后挪一天**——今天算出的信号，明天才据此交易。代码里体现为 `signal.shift(1)`。这是回测可信度的生命线。

---

## 三、回测引擎在做什么？

回测 = 拿历史数据，逐日模拟「按这个策略操作，账户净值如何变化」。

我们的最小引擎逻辑：
1. 拿到每日「目标仓位」（0 或 1，已 shift 过）。
2. 计算每日收益：持仓时 = 当日价格涨跌幅；空仓时 = 0。
3. **扣成本**：每次仓位发生变化（买入或卖出）时，按成交额扣**手续费 + 滑点**。
4. 累乘得到**净值曲线**（净值=1 起步，代表 1 元本金的增长轨迹）。

**纳入的真实约束**：
| 约束 | 我们怎么处理 | 为什么 |
|------|------------|--------|
| 手续费 | 双边按比例扣（如万 2.5）| 频繁交易会被手续费吃掉收益 |
| 滑点 | 成交价比理想价差一点 | 真实成交不可能刚好在收盘价 |
| T+1 | 信号 shift(1)，今买明卖 | A股当天买入不能卖 |
| 涨跌停 | M2 先简化不处理，M3 再加 | 先跑通主干，再加细节 |

## 四、四个绩效指标（这次亲手算）

| 指标 | 公式（直觉） | 说明 |
|------|------------|------|
| **总收益率** | 期末净值 / 期初净值 − 1 | 整段时间一共赚了多少 |
| **年化收益率** | (期末/期初)^(252/天数) − 1 | 折算成「一年能赚多少」，便于横向比。252=A股年均交易日 |
| **最大回撤** | max(1 − 净值/历史最高净值) | 最痛的一段从高点跌了多少 |
| **夏普比率** | 日均超额收益 / 日收益标准差 × √252 | 每承担一份波动，换来多少收益。越高越「稳赚」 |

> 我们会同时算「策略」和「买入持有(Buy&Hold)」两条曲线对比——如果策略跑不赢「一直拿着不动」，那它就没创造 Alpha。**对照基准**是判断策略好坏的关键。
