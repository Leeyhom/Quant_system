# 13 · 多因子 IC 加权合成：把「稳定 IC」分散成「稳定超额」

> 接 `12`。M12 把价值因子中性化后，证明了它有**干净且统计稳定**的选股力（ICIR 0.30→0.39、OOS test t 大幅提升），但也留下一个诚实的 nuance：**单因子躲不过 regime 依赖**——2025Q3 行业级回调时，连中性化后的价值因子单季 IC 都是负的。本章要回答：**把多个已中性化、OOS 同号、彼此低相关的因子合起来，能不能把"某一个因子在某段行情失灵"的风险分散掉？**

---

## 一、为什么合成能分散 regime 依赖

单因子的命门是「它什么时候有效」由市场风格（regime）决定，而风格不可控：

- 价值因子（earnings_yield）在「价值占优」的行情里强，在「成长/动量占优」的行情里弱甚至反向。
- 流动性因子（amihud）、动量因子各有自己的 regime。

这些因子的失灵时点**不完全同步**——价值熄火那一季，动量或流动性可能正好顶上。把它们合成一个组合因子，就像把多个**正期望、低相关**的赌注放进一个篮子：单个分量的方差被部分抵消，合成 IC 的**波动（std_ic）下降**，于是 **ICIR = meanIC / stdIC 上升**，信号更平稳。

⚠️ 关键前提是**低相关**。如果两个因子高度相关（比如 1/PE、1/PB、1/PS 三个价值因子彼此相关常 >0.7），合在一起只是把同一个赌注下三遍，分散不了任何风险，还白白稀释了其他正交信息。所以合成前**必须先看相关矩阵**，把冗余的同类因子去掉或只留一个代表。

---

## 二、为什么用 IC 加权，而不是等权

等权合成（M9 的 `combine_factors`：各因子 rank 后取平均）把每个因子当成同样可信。但因子的预测力天差地别——一个 OOS meanIC 0.06 的因子，和一个 0.01 的因子，凭什么一样权重？

**IC 加权**：让每个因子的权重正比于它的历史 IC（预测力）。IC 大的因子说话更响，IC 接近 0 的因子近乎沉默。

```text
weight[因子k] = IC[因子k] / Σ|IC[所有因子]|        （IC 已带符号，可定向）
composite = Σ weight[k] · rank(因子k)
```

直觉：这就是「按过去的命中率下注」。它比等权更贴近「信息量 = 预测力」的本意，也比让人手调权重更不容易过拟合（权重由数据定，不是我们猜的）。

> 注：更"正统"的做法是 IC_IR 加权（除以 IC 波动）甚至带协方差的最优化（Markowitz 式）。本章先用最朴素的 **|IC| 归一化加权**，把"权重由数据定、防前视"这条主干打通；ICIR 加权留作后续增量，避免一上来就引入需要估协方差的复杂度（信息量优先于模型复杂度）。

---

## 三、防前视：权重只能用「过去」的 IC

这是整个合成里最容易作弊、也最致命的地方。

如果我用**全样本** IC 算权重，再回到样本里去验证「合成因子多好」——那是拿考试答案反推该重点复习哪科，必然虚高。权重里偷看了未来。

**铁律：算权重用的 IC，只能来自切分日 `upto_date` 之前的数据。** 复用 M9 `build_oriented_composite` 已经验证过的口径：

1. 只取 `fwd_ret.index < upto_date` 的未来收益。
2. 再裁掉这段末尾 `horizon` 天——因为第 t 日的 IC 用到 t→t+horizon 的未来收益，最后 horizon 天会「探入」test 段（见 docs/10）。
3. 用裁剪后的 train 段 IC 定**方向**（IC<0 取负）**和权重大小**（|IC| 归一化）。
4. test 段只是拿这套**冻结的权重**去算合成因子，绝不重算。

定向（`orient_by_ic`）和权重共享同一个 train IC，天然一致：方向是 IC 的符号，权重是 IC 的绝对值大小。

## 四、和现有积木怎么拼

M13 不另起炉灶，全部踩在已验证的模块上：

| 步骤 | 复用的模块 |
|------|-----------|
| 各分量先中性化（剥行业/市值 beta） | `quant/factor/neutralize.py`（M12） |
| 算 train 段 IC（定向 + 权重） | `daily_ic` / `ic_summary`（M7）、`orient_by_ic`（M9） |
| 合成（rank 后按权重线性组合） | 新增 `ic_weighted_composite`（本章），等权对照用 `combine_factors`（M9） |
| 合成因子的 OOS IC 是否更稳 | `ic_train_test`（M9） |
| 合成因子分层多头是否跑赢基准 | `long_top_layer`（M12） |
| 滚动 walk-forward 稳定性 | `rolling_walk_forward` / `stability_summary`（M7） |

新增的只有薄薄一层 `composite.py`：`factor_correlation`（相关矩阵去冗余）和 `ic_weighted_composite`（权重只用过去）。

## 五、怎么算「成功」

M13 的成功标准不是「合成因子收益更高」（那可能又是某段 regime 的运气），而是**稳定性**：

1. **OOS IC 更稳**：合成因子的 test 段 ICIR / |t| ≥ 最好的单因子，且 std_ic 下降（分散见效）。
2. **滚动跑赢比例更高**：IC 加权合成的分层多头，在滚动 walk-forward 上「跑赢等权基准的窗口比例」高于 M12 的单因子。
3. **IC 加权 ≥ 等权**：若 IC 加权并不优于朴素等权，要诚实指出——可能是分量太少、相关太高，或这段样本里预测力排序本身不稳。

如果三条都达成 → 这是把「稳定 IC」兑现成「稳定超额收益」的关键一步，可以走向组合资金管理 / 模拟盘。若否 → 诚实记录（可能需要更长历史、更多正交因子、或更大池子），不强行宣布胜利。

## 六、实跑结果：一个诚实的负结果（必须记下来）

在本项目 89 股全池、2024~2025 这段数据上，`scripts/composite_demo.py` 跑出来的结论是**合成没有跑赢事后最优的单因子**：

| 指标（test 段 / 滚动） | 最优单因子(amihud) | 等权合成 | IC加权合成 |
|------|------|------|------|
| test ICIR | **+0.69** | −0.31 | −0.22 |
| 滚动跑赢基准比例 | **75%** | 50% | 50% |

为什么合成反而更差？根因有两条，都很有教育意义：

1. **单因子 IC 符号在切分点附近极不稳。** amihud、momentum20、book_to_price 在 train 段和 test 段**变号**（train 为正、test 为负或反之）。IC 加权的权重**方向**是用 train IC 的符号定的——一旦这个符号在 test 段翻转，等于「按昨天的方向下了今天反向的注」，合成把几个方向都押错的因子加在一起，自然劣于那个恰好没翻车的单因子（amihud 的 test ICIR 是事后才知道高的，train 段未必选得到它）。
2. **分量信息冗余。** 5 个候选里 earnings_yield / sales_yield / book_to_price 是价值同类（相关矩阵里 sales-book 相关 0.53），真正正交的只有 amihud、momentum 两个。靠堆同类因子凑数量，分散不了 regime 风险——这正好印证了第一节强调的「低相关是前提」。

**这说明 M13 的朴素 |IC| 加权还不够。** 它对**单点切分**的 train IC 太敏感：权重（尤其方向）由切分日前那一段行情决定，而那段行情未必代表 test。教训写进下一步：

- **ICIR 加权而非 |IC| 加权**：除以 IC 波动，惩罚那些「均值大但飘忽」的因子，降低对单段行情的敏感。
- **多切分点平均权重**：不在单一 `upto_date` 定权重，而是滚动多个切分点取平均，让权重更稳。
- **找真正正交的因子**，而不是加同类因子的数量——信息量来自正交性，不来自因子个数。

> 诚实优先于好看的结论：合成这一步在当前数据上没赢，但失败的**方式**（IC 符号不稳、分量冗余）恰恰指明了改进方向。这比一个侥幸跑赢的结果更有价值。

## 七、打磨 1：ICIR 加权 + 多切分点平均权重

针对第六节暴露的两个弱点，做了两处增量（`weighted_composite`）：

**① ICIR 加权**（`scheme="icir"`）：权重 ∝ `meanIC / stdIC` 而非 `|meanIC|`。同样的均值预测力，**波动越大的因子权重越小**——惩罚那些"均值好看但每天忽强忽弱"的因子，降低对某一段行情的敏感。

**② 多切分点平均**（`n_cuts>1`）：不在单一切分日 `upto_date` 定权重，而是在 train 区间后半段取多个 expanding 切分点，**各算一版带符号权重再平均**。带符号是关键——若某因子在不同切分点 IC 变号，正负相消，其平均权重自动趋于 0，等于把"方向不稳"的因子**收缩掉**，而不是盲目押某一段的方向。

两处都严格防前视：所有切分点都 < `upto_date`，每个切分点的 IC 都裁掉末尾 horizon 重叠。

实跑对比（ICIR×多切分 vs 第六节的 |IC| 单点基线）：

| | limit20 子池 | 全池 89 只 |
|------|------|------|
| test ICIR：\|IC\|单点 → ICIR×多切分 | +0.21 → **+0.32** | −0.22 → **−0.16** |
| 滚动跑赢基准比例（ICIR×多切分） | 75% | 50% |
| 全样本分层多头夏普（ICIR×多切分 / 单因子 / 基准） | **1.91** / 0.69 / 0.84 | 0.68 / 0.98 / 0.87 |

**两个池子里 ICIR×多切分都稳定优于 |IC| 单点基线**（test ICIR 都更高），证明打磨方向对：降噪 + 多切分确实让权重更稳。子池上甚至大幅跑赢单因子和基准（夏普 1.91）。

## 八、打磨 2：但全池仍未翻盘——根因不在加权，在信息量

全池上 ICIR×多切分的合成 IC 仍是**负的**（−0.16），还是输给事后最优单因子 amihud（+0.69）。为什么加权技巧救不回来？看全池 test 段单因子 IC：

```
earnings_yield  test ICIR +0.62  同号 ✅
sales_yield     test ICIR -0.07  变号
book_to_price   test ICIR -0.55  变号
amihud          test ICIR +0.69  变号（train 为负，test 才转正）
momentum20      test ICIR +0.19  变号
```

**5 个因子里 4 个在 70/30 切分点处 train→test 变号**——这正是 M12 诊断出的 2025Q3 行业级 regime 断点。问题的本质是：

- 加权（无论 |IC| 还是 ICIR）的**方向**都来自 train 段 IC 的符号。当一个因子在 test 段相对 train 段**整体翻号**时，多切分平均也救不了——因为所有切分点都在 train 段内、都还没"看到"这次翻号，平均出来的方向依然是 train 的旧方向。多切分能压住的是**切分点之间的抖动**，压不住**train 与 test 之间的整体断层**。
- 真正同号稳定的只有 earnings_yield 一个。其余 4 个把噪声（甚至反向信号）灌进合成，自然把它拖到负 IC。**5 个候选里真正正交又稳定的信息其实只有 1～2 份**（相关矩阵里价值三兄弟挤在一起，amihud/momentum 各 1 份但 OOS 不稳）。

**结论（诚实）：打磨成功了它能解决的问题（加权稳健性，ICIR×多切分确实 > |IC|单点），但没解决它解决不了的问题（信息量不足 + regime 整体断层）。** 加权是"把已有的稳定信号更好地组合"，它无法**凭空制造**稳定信号。当池子里 OOS 同号的因子只有一个时，任何合成都只是在稀释这唯一的好信号。

## 九、下一步：从"调加权"转向"补正交信息"

加权这条路已经走到边际收益递减。真正的瓶颈是**正交且 OOS 稳定的因子太少**，方向应是：

1. **补正交因子**：当前 5 个里价值占 3 个（冗余）。需要的是与价值低相关、且 OOS 同号的新维度——质量（ROE/毛利率稳定性）、成长（营收/利润增速）、分析师预期修正等。信息量来自**正交性**，不来自因子个数。
2. **扩池 + 拉长历史**：89 只 / 2 年只够覆盖 1～2 个 regime，撞上 2025Q3 一次断层就被带偏。更长历史能让"哪些因子真稳"的判断不被单次 regime 主导。
3. **regime 自适应**（更高级，暂不做）：识别当前市场风格，动态切换因子方向/权重——但这要先有足够多稳定因子打底，否则只是过拟合 regime 标签。

> 一句话：M13 把"多因子合成"这条工程链路（相关去冗余 → 防前视加权 → ICIR降噪 → 多切分稳健 → OOS验证）完整打通且验证有效，但也用数据证明了**当前阶段的真正约束是信息量而非模型**——回到项目第一性原理「信息量优先于模型复杂度」。下一步补正交因子，而不是继续雕加权。

---

## 十、一句话回顾

> 单因子的命门是 regime 依赖；合成多个**低相关、OOS 同号**的因子能分散它。用 **|IC| 加权**让预测力强的因子说话更响，权重**只用切分日之前的 IC**算（防前视），方向与权重共享同一段 train IC。评判看的是 **OOS IC 稳定性**和**滚动跑赢比例**，不是单段收益。
