## 一、量化交易到底在做什么？

传统炒股靠「感觉」和「看新闻」。**量化交易**则是：把交易决策变成**明确的规则**，用历史数据**验证**这些规则是否真的赚钱，然后让程序**自动或半自动**执行。

一个量化系统的完整生命周期：

```
数据获取  →  策略开发  →  回测验证  →  模拟/实盘  →  监控与风控
（M1）        （M2）        （M2）       （M3）         （M3）
```

我们现在在 **M1：数据获取**。没有干净、可靠的数据，后面一切都是空中楼阁——这就是为什么我们从数据管道开始。

## 二、四个必须建立直觉的概念

| 概念 | 一句话理解 | 为什么重要 |
|------|-----------|-----------|
| **收益率 (Return)** | 赚了百分之多少 | 衡量「赚多少」 |
| **最大回撤 (Max Drawdown)** | 从最高点跌到最低点，最多亏了百分之多少 | 衡量「最难受时有多痛」，决定你拿不拿得住 |
| **夏普比率 (Sharpe Ratio)** | 每承担一份波动风险，换来多少超额收益 | 衡量「性价比」，光收益高不算好，得稳 |
| **Alpha / Beta** | Beta=跟大盘一起涨跌的部分；Alpha=靠本事多赚的部分 | 量化追求的是 **Alpha**——能跑赢市场的真本事 |

直觉例子：A 策略一年赚 30% 但中途一度亏 50%；B 策略一年赚 15% 但最多只亏 8%。
很多人扛不过 A 的 50% 回撤会中途割肉离场，所以 **B 的夏普比率更高，往往是更"可用"的策略**。这就是为什么我们不只看收益率。

> 这些指标的**具体公式和代码实现**会放到 M2 回测引擎里，那时你会亲手算出来，现在只需建立直觉。

## 三、A股市场微观结构（为什么先做A股）

A股规则统一、数据免费易得，适合作为第一个完整闭环来学。但它有几条**独特规则**，会直接影响策略和回测，必须先知道：

### 1. T+1 交易制度
- **今天买入的股票，最早明天才能卖。**
- 影响：策略里不能「当天买当天卖」，回测引擎必须强制这条约束，否则回测收益是**假的**（M2 会实现）。

### 2. 涨跌停限制
- 主板：单日涨跌幅限制 **±10%**；
- 创业板/科创板：**±20%**；
- ST 股：**±5%**。
- 影响：涨停时你想买**买不进**，跌停时你想卖**卖不出**。真实回测必须模拟这种「成交失败」，否则又是假收益。

### 3. 复权（前复权 qfq / 后复权 hfq）
- 公司分红、送股会让股价在某一天「凭空跳水」，但这不是真的下跌。
- **前复权**：保持最新价真实，把历史价往下调，使价格序列连续可比。
- **为什么用前复权**：算均线、动量等技术指标时，连续的价格才不会产生**假信号**。这就是 `fetch_daily(..., adjust="qfq")` 默认前复权的原因。

> 港股、美股是 **T+0**（当天可买卖）、**无涨跌停**（美股有熔断但机制不同），规则更简单。等 A股闭环跑通后，加入它们主要是「换数据源 + 调规则参数」，架构不变。

## 四、数据管道在系统中的意义

我们这一步搭的 `akshare_loader` 做了三件事，对应三个工程原则：

1. **统一格式**：akshare 返回中文列名，我们在入口翻译成英文标准列（date/open/close...）。
   → 上层策略代码**不依赖**数据源细节，将来换 tushare / 换市场只改这一层。
2. **本地落地 (Parquet)**：拉一次存下来，反复回测时直接读本地，**又快又不被数据源限流**。
   → Parquet 是列式存储，比 CSV 体积小、读写快、还保留数据类型。
3. **可复现**：同样的代码 + 同样的本地数据 = 同样的结果，这是科学验证策略的前提。
