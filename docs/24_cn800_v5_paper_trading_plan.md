# 24 CN800 v5 模拟盘观察计划

> 当前状态：`scripts/joinquant_cn800_strategy_v5.py` 已接入聚宽模拟盘。本文档定义观察期、数据回收口径和下一轮迭代门槛。

## 1. 目的

这次模拟盘不是为了立刻证明 v5 有稳定 alpha，而是为了验证三件事：

1. 回测策略在真实平台调度下能否按预期生成持仓。
2. 交易执行是否受现金、整手、停牌、涨跌停、滑点和手续费影响。
3. 模拟盘路径是否暴露出历史回测看不到的冷启动和状态依赖问题。

一个月样本太短，不能据此大幅优化因子权重或参数。它主要用于发现执行差异和数据口径问题。

## 2. 当前主线

```text
scripts/joinquant_cn800_strategy_v5.py
```

v5 的核心构成：

- CN800 股票池：沪深300 + 中证500 扩展池。
- 7 因子：`value_blend`、`growth_peg`、`amihud`、`quality_roe`、`low_vol_60`、`residual_momentum`、`roe_stability`。
- 风控/组合：波动率目标降仓、自适应动量、动态行业上限、资金效率优化。

历史定位：

- v9/v10 是 CN800 之前的聚宽历史基线和冷启动经验，不是当前模拟盘主线。
- v4 是 CN800 轻量强候选，v5 是当前接入模拟盘的生产候选。

## 3. 一个月后需要回收的数据

建议目录：

```text
jointquant/cn800_v5_paper/
```

需要保存或记录：

- 聚宽模拟盘设置：开始日期、结束日期、初始资金、基准、手续费/滑点设置。
- 交易导出：`transaction.csv`。
- 持仓导出：`position.csv`。
- 日志导出：`log.txt`。
- 收益截图或净值序列。
- 观察期内是否人工干预过策略。

原始 CSV 和 zip 默认不入 git；分析后的 markdown/csv 摘要可以入库。

## 4. 复盘指标

回收后至少输出：

- 收益：策略收益、基准收益、超额收益。
- 风险：最大回撤、日波动、单票最大权重。
- 执行：平均现金、平均持仓数、换手、费用、滑点近似。
- 异常：停牌、涨停买不进、跌停卖不出、整手导致的仓位偏差。
- 路径：前几次调仓目标、实际成交、持仓延续情况。
- 对照：与同日期回测或目标持仓的差异。

## 5. 观察期纪律

观察期内默认不做以下事情：

- 不因为一两次盈亏修改因子权重。
- 不因为一个月跑赢就宣布策略可实盘。
- 不因为一个月跑输就否定长期因子。
- 不把模拟盘账户状态和全周期历史回测混成同一个证据。

可以做的事情：

- 修日志解析、导出分析和文档。
- 记录订单失败和现金闲置。
- 检查是否有未来函数、股票池前视或平台数据口径差异。
- 准备下一轮消融实验。

## 6. 下一轮决策门槛

一个月后根据复盘结果选择下一步：

| 现象 | 优先动作 |
|---|---|
| 持仓/调仓和预期不一致 | 先修执行和日志，不改 alpha |
| 现金长期过高 | 检查整手、单票价格、`MAX_SLOT_MULT` 和目标权重 |
| 个别行业/个股贡献极端 | 做行业/个股归因，不直接追热点 |
| 回撤超出历史同阶段 | 检查仓位、动量过滤和市场 regime |
| 模拟盘稳定但样本太短 | 继续观察，补本地动态池和消融验证 |

真正进入下一版策略前，必须重新回答：

1. 改动解决的是 alpha、执行、风控还是数据问题？
2. 是否有样本外或真实导出证据？
3. 是否引入未来函数、幸存者偏差或全样本择优？
4. 是否值得接入模拟盘，而不是只留作研究实验？
