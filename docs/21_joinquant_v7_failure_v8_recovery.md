# 21 · 聚宽 A股 v7 失败复盘与 v8 稳健恢复

> v7 的聚宽真实表现显著差于 v6：总收益从 v6 的 +188.42% 降到 +111.13%，
> 最大回撤从 22.28% 放大到 34.01%。这不是“正常抖动”，而是一次有效的失败样本。

v7 的几个改动叠加后产生了坏组合：

- 取消 `mom120 > -10%`，让更多低估值下跌股进入组合。
- `REBALANCE_DAYS=50` 改变了调仓日路径，2021 回撤开始前更早切入弱势票。
- `SCORE_TILT=0.65` 在本地看是轻倾斜，但 6 万资金 + 100 股整手后，平均最大单票权重从 10.58% 升到 12.16%。
- 本地验证使用的是 `DEFAULT_POOL` 的 89 只有效票，而聚宽实际跑的是 152 只候选，横截面排名不是同一个问题。

所以 v7 的核心错误不是某个单一参数，而是“用不完全一致的数据池做 in-sample 搜索，再把搜索出的组合施工方式迁移到聚宽真实池”。

## 三、是否过拟合

是，v7 有明显过拟合。

证据：

1. v7 的默认参数来自 2019~2025 本地全样本排序，而不是 walk-forward 后的独立测试。
2. 本地验证池与聚宽真实池不一致：本地 `DEFAULT_POOL` 有效约 89 只，聚宽候选约 150 只。
3. 目标函数偏向 Sharpe/收益排名，缺少“参数稳定性”和“真实聚宽导出一致性”约束。
4. `q00_lv00/top8/fixed98` 一类候选本地收益很高，但回撤也高，说明搜索已经开始偏向进攻暴露。
5. v7 实盘结果 2019~2024 连续输给 v6，只在 2025 追上，典型后段样本适配。

## 四、是否有未来函数

目前没有发现明确未来函数。

聚宽策略里：

- `get_fundamentals(q, date=context.previous_date)` 用上一交易日数据。
- `get_price(..., end_date=context.previous_date, count=140)` 用上一交易日前的历史价格。
- 调仓在当日 14:30，根据上一交易日及以前数据打分。
- 动量、波动、流动性都来自历史窗口。

更准确的判断是：v7 不是“未来函数作弊导致回测虚高”，而是“本地验证和聚宽真实环境不一致 + 参数选择过拟合”。

## 五、是否需要更多数据

需要，而且优先级很高。

当前 v7 产生过程主要依赖：

- 本地因子数据：2018-01 起。
- 本地回测窗口：2019-01-01 ~ 2025-12-31。
- 本地有效股票池：`DEFAULT_POOL` 约 89 只。
- 聚宽真实策略池：约 152 只。

先做两件事：

1. 把聚宽策略池 152 只完整拉到本地缓存：

```bash
NO_PROXY='*' PYTHONPATH=. python scripts/refetch_joinquant_pool.py
```

2. 重新跑本地验证，必须用和聚宽一致的股票池。

关于时间拉长：本地 Akshare 估值接口实际可靠起点通常是 2018-01，所以本地研究很难无缝拉到 2015。聚宽端可能能跑更早，但若本地无法复现，就不能用它直接调参。更合理的做法是：

- 本地：2018~2025，用完整 152 池做 walk-forward。
- 聚宽：如果能回测到 2015，作为额外压力测试，不用来直接选参数。

## 六、AI-Trader 能否帮到我们

可以帮，但不是以“直接给 A股选股因子”的方式。

`HKUDS/AI-Trader` 的 README 把它定位成 agent-native trading platform，重点是代理接入、信号同步、copy trading、挑战评测和跨市场交易信号。它的技能文件也显示核心能力是注册 agent、读取信号流、发布策略/交易、参加 challenge，并不是一个 A股基本面因子库或回测引擎。

论文摘要里更重要的启发是：通用智能不自动等于交易能力，风险控制决定跨市场稳健性。这和 v7 的失败正好一致：不是模型/智能不够，而是验证闭环和风控约束不够硬。

可借鉴：

- 把每个策略版本当成 challenge variant 记录，不只看总收益，也记录 max drawdown、risk-adjusted score、trade_count。
- 做 agent 讨论/复盘，但最终必须落到可执行的规则和可复现数据。
- 引入外部信号平台时，只作为“候选信息源”，不能直接覆盖本地因子和风控。

暂不建议：

- 直接把 AI agent 的实时观点接入 A股实盘交易。
- 用 copy trading 替代我们当前的因子研究。
- 在没有数据对齐之前引入 LLM 选股模型。

## 七、v8 决策

v8 不追求“再创新高”，而是先恢复到真实有效的结构：

- 从 v6 继承：`TOP_N=10`、`INDUSTRY_CAP=2`、`REBALANCE_DAYS=60`、`MAX_EXPOSURE=95%`。
- 恢复 `mom120 > -10%` 风险过滤。
- 取消 v7 的分数倾斜，回到等权。
- `INCLUDE_HOLDER=False`，诚实对齐聚宽真实覆盖。

产物：

```text
scripts/joinquant_cn_sim_strategy_v8.py
scripts/refetch_joinquant_pool.py
```

执行顺序：

1. 先跑 `joinquant_cn_sim_strategy_v8.py`，它应接近 v6 真实表现。
2. 同时补齐聚宽 152 池本地缓存。
3. 只有当本地验证池和聚宽池一致后，再做 v9 的收益增强。

这次的核心教训：在数据未对齐前，最优参数不是 alpha，而是噪声。
