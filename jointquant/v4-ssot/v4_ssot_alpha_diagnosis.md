# v4-ssot Alpha 诊断

日期：2026-06-29  
对象：`scripts/joinquant_cn_sim_strategy_v4_ssot.py` + `jointquant/v4-ssot/` 聚宽导出

## 结论

v4-ssot 不是“没有 alpha”，而是 alpha 密度不够高，且组合构造仍然像一个低估值/低流动性轮动策略。2019-01-02 到 2025-12-31 的聚宽导出显示：

- 总收益：+123.18%，年化约 +12.65%（聚宽截图为 +12.54%）
- 聚宽截图口径：超额收益 +45.13%，alpha 0.067，beta 0.741，夏普 0.511，最大回撤 27.09%
- 本地按日净值重算：最大回撤 27.09%，最深回撤发生在 2021-01-13 到 2024-01-22
- 平均仓位：85.62%，平均现金：14.38%，平均持股：9.91 只
- 调仓 43 次，订单 479 笔，费用 3180.61 元，卖出胜率 65.91%
- 每次调仓目标名单平均换手 42.38%，成交额口径平均毛换手 75.31%

真正的问题：2021-2023 三年几乎没有恢复能力，收益主要靠 2019、2020、2024、2025 的市场和风格窗口。这不适合直接实盘放大。

## 根因排序

### P0：行业映射没有覆盖扩大的股票池

v4-ssot 的 `STOCK_POOL` 有 152 只，但当前 `INDUSTRY_MAP` 只覆盖 114 只，38 只（25.0%）落入“其他”：

```text
000008, 000039, 000100, 000338, 000776, 002027, 002236, 002371, 002415,
002493, 002821, 002916, 003816, 300017, 300124, 600346, 600438, 600660,
600873, 600905, 600989, 601138, 601298, 601319, 601877, 601888, 601898,
603893, 603986, 688009, 688036, 688111, 688169, 688256, 688289, 688303,
688561, 688981
```

这会同时伤害两件事：

- 行业中性化：把汽车、电子、新能源、医药、周期等混在“其他”里回归，残差不再是干净的行业内 alpha。
- 行业上限：`INDUSTRY_CAP=1` 时，“其他”也只能选一只，变成一个没有经济意义的超级行业槽位。

这是当前最应该先修的结构性 bug。

### P1：当前跑的是 5 因子对齐版，不是 6 因子收益版

策略文件写着 v4 六因子，但实际参数为：

```python
INCLUDE_HOLDER = False
```

聚宽日志也确认最终只跑：

```text
earnings_yield,cashflow_yield,sales_yield,growth_peg,amihud
```

本地 `results/cn_holder_factor_eval.csv` 显示，加入筹码集中度后，5 因子加执行约束从 +141.08% / Sharpe 0.792 / 回撤 27.73%，提升到 +158.65% / Sharpe 0.856 / 回撤 29.08%。它不是银弹，但至少是已验证的正交增量信号。当前为了“对齐验证”关掉它，和“收益最大化”目标冲突。

### P1：三个价值因子重复加权，信息量不够宽

`1/PE、1/PCF、1/PS` 同属便宜/价值暴露，各占 20% 权重，等于价值被重复投票三次。实际收益贡献也符合这个判断：交通运输、银行、建筑、有色周期、非银、家电贡献较多；电子、地产、新能源设备、传媒拖累。

这解释了为什么它在 2021-2023 这种价值陷阱/风格切换期恢复慢。

### P2：小资金 10 只组合有现金拖累

6 万本金、10 只、100 股整手约束下，平均现金 14.38%，显著高于目标现金 5%。这不是致命问题，但会吞掉一部分年化收益。v4 旧版平均现金 9.25%，一个原因是持股更集中，单票目标金额更大，更容易凑整手。

### P2：换手不低，但信号更新不够强

每次调仓目标名单平均换手 42.38%，成交额口径平均毛换手 75.31%。费用只占最终利润约 4.3%，不是第一矛盾；第一矛盾是换手买来的新名单没有足够强的增量 alpha。

## 开源项目给的启发

- Microsoft Qlib 把量化链路拆成 data processing、model training、back-testing，并覆盖 alpha seeking、risk modeling、portfolio optimization、order execution。这说明下一步不该只调因子权重，而要把风险模型和组合优化纳入同一流程。
- Alphalens 的核心用途是预测性股票因子的 performance analysis，强调 factor/forward return 清洗、分组和 tear sheet。你的项目已经有 IC，但生产策略缺少每次调仓后的分组归因、行业归因和因子暴露报告。
- `ml-quant-trading` 这类多因子项目把因子库扩到数百维，但关键不在数量，而在 mask-aware、分组、筛选和偏差控制。对当前项目而言，下一阶段应该“宽因子库 + 严格 walk-forward 筛选”，而不是继续在 5 个因子里微调。

## 下一版建议

### v6 方向：先修风险模型，再加正交信号

1. 补全 v4-ssot 152 只池子的行业映射。短期可复用 `quant/data/industry.py` 的 DEFAULT_POOL 映射并手工补 38 只；中期应切到申万/中信行业。
2. 打开 `INCLUDE_HOLDER=True`，但只在确认聚宽字段真实有覆盖后启用；如果字段覆盖低，必须在日志里输出 holder 覆盖率。
3. 把价值三因子合成一个 `value_blend`，避免 `1/PE、1/PCF、1/PS` 三票重复投票。
4. 增加正交信号：`quality_roe` 半权重、`low_vol_60` 半权重、`momentum_120` 只做过滤不做追涨。
5. 使用 v5 的动量过滤，但不要退回旧成长口径：保留 `growth_peg`，过滤规则建议先用 `mom120 > -10%` 或 `not_deep_downtrend`。
6. 调仓从固定 40 日改成“40 日基础 + 分数滞后阈值”：新股分数不明显超过旧持仓时不换，降低无效换手。
7. 对 6 万本金单独优化持股数：Top 8 可能比 Top 10 更合适，因为整手约束下现金拖累更低。

### 严格验证标准

下一版不要只看全样本收益。必须输出：

- 2019-2025 全期聚宽回测：收益、alpha、beta、夏普、最大回撤
- 逐年收益，特别是 2021、2022、2023 是否改善
- 行业暴露和行业贡献
- 每次调仓目标名单留存率和毛换手
- 因子覆盖率：PEG、PCF、PS、holder、quality 的非空比例
- 与 v3/v5/v4-ssot 同口径对比

## 交易员判断

当前不应该继续在 v4-ssot 上“微调参数”。它的主要瓶颈不是 `TOP_N=10` 或 `REBALANCE_DAYS=40`，而是：

1. 行业分类错误污染了中性化；
2. 收益版因子被关掉；
3. 价值信息重复、正交信息不足；
4. 没有市场/个股状态过滤来避开价值陷阱；
5. 小资金整手约束造成现金拖累。

优先级是：修行业映射 -> 恢复筹码/质量/低波这些正交因子 -> 用趋势过滤控制价值陷阱 -> 再谈更复杂的 ML 或动态权重。
